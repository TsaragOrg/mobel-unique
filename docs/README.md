# Documentation

This directory contains project documentation that should evolve with the product.

## Main Areas

- `specs`: product, technical, and feature specifications.
- `decisions`: architecture decision records.
- `plans`: execution plans linked to accepted specs.
- `roadmap`: package-level work tracking.
- `quality`: review and verification standards.
- `workflow`: development workflows.

Keep specs close to implementation. When a feature changes behavior, permissions, data, or deployment assumptions, update the relevant document before or during the code change.
