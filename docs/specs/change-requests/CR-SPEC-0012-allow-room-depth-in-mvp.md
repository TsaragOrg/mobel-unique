# CR-SPEC-0012 Allow Room Depth In MVP

Target spec ids: SPEC-0012
Related draft specs: SPEC-0015
Status: proposed

## Reason For Change

`SPEC-0012` Step 7 currently states that the MVP must not ask for room depth,
camera distance, or extra free-form room measurements. The intent was to keep
mobile dimension entry minimal.

In practice, the in-home simulation worker placement pipeline benefits
materially from a known room depth. The `OpenAIPlacementProvider` and
`OpenAIPlacementMeasurementProvider` use depth to scale the sofa correctly in
perspective. Without depth, the model must infer it from visual cues and
placement accuracy degrades for some room geometries.

The product owner has decided that one extra mandatory field on the dimension
form is an acceptable trade-off for materially better visualization quality.

## Proposed Change

Update `SPEC-0012` Step 7 (Dimension Guide And Dimension Entry) so that:

- For `back_wall` mode, required inputs are: `wall_width`, `wall_height`,
  `room_depth`.
- For `corner` mode, required inputs are: `left_wall_width`,
  `right_wall_width`, `room_height`, `room_depth`.
- The constraint "the MVP must not ask for room depth" is removed.
- The dimension guide image continues to indicate exactly which dimensions to
  provide. Room depth is presented as the green guide line ("profondeur").
- All depth inputs are validated as positive numbers in metres with reasonable
  upper bounds.

## Impact

- Public UI: dimension form gains one mandatory field. The guide image
  generated by the worker already labels depth via the green guide line.
- API: `POST /api/public/simulations/{simulation_job_id}/dimensions` request
  body adds `room_depth` to required fields for both modes.
- Worker: `lib/dimensions.ts` validation accepts `room_depth` as required,
  not optional. `OpenAIPlacementProvider` and
  `OpenAIPlacementMeasurementProvider` consume the depth when building prompts.
- Database: `supplied_dimensions` jsonb shape gains a required `room_depth`
  key. No migration needed (jsonb is schemaless).
- Tests: existing dimension validation tests must be updated to require
  `room_depth`.
- Roadmap: no changes needed.
